STATE MANAGEMENT

  • On reactive programming components interact each other as soon they change their states
  • A set of observable collections and variables help us to notify about these changes and proceed as needed
  • But it's difficult to keep them tracked once the application starts growing up
  • Akita comes to help us to manage these interations. (think about a DATABASE for UI)
  • Built on top of RxJS (multiple data stores from Flux and the immutable updates from Redux)
  • Simple and effective
  • Pattern known as 'Observable Data Stores'
  • POO principles

BASIC CONCEPTS

Model (Entity principle) Entity Store (single source of truth) Entiy Query (layer between components and entity store)
